# Priority Rules

Rank topics by expected score gain under time pressure, not by chapter order.

## Core Formula

Use this reasoning qualitatively:

`priority = exam likelihood * point value * learnability * prerequisite value * material evidence`

Do not present this as a precise numeric calculation unless the user asks for a scoring model.

## Evidence Order

Prefer evidence in this order:

1. Teacher-highlighted or explicitly required content
2. Past papers, sample papers, question banks, repeated question types
3. Assignments, lab reports, classroom examples, review exercises
4. Lecture slides with repeated emphasis, long coverage, summaries, or worked examples
5. Core concepts needed by many later topics
6. Ordinary slide details
7. Supplementary reading and low-frequency extensions

## Priority Buckets

- Must rescue: likely to appear, high value, foundational, or repeatedly emphasized.
- High yield: good score return for limited time.
- If time remains: useful but not urgent.
- Skip for now: low evidence, low value, or too costly for current time.

## Important Heuristic

The best sprint target is often "partly familiar but unstable", not "completely unknown".

Upgrade topics the student almost knows and can stabilize quickly. Downgrade topics that are completely unknown, difficult, low-frequency, and not foundational.

## Type-Specific Adjustments

Calculation and derivation:
High-frequency problem types > standard steps > formula conditions > common mistakes > proofs > rare extensions.

Conceptual understanding:
Core framework > concept comparisons > process diagrams > short-answer templates > minor definitions.

Memorization and recall:
Essay frames > frequent short answers > keywords > small facts > background material.

Practical skills:
Complete task flow > required deliverables > common errors > sample reproduction > theory explanation.

Language and expression:
Question types > templates > high-frequency vocabulary > timed practice > advanced expression.

Open-ended synthesis:
Answer framework > reusable cases > material index > argument evidence > polishing.
