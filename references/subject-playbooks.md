# Subject Playbooks

Use this after classifying the course with `course-types.md`. The playbook controls what "helpful" means for the first rescue pack.

## Calculation And Derivation

Examples: probability, calculus, linear algebra, physics, circuits, mechanics.

Default rescue artifacts:
- symbol decoder
- formula condition card
- question-type templates
- one worked micro-example
- exact exercise list or mini drill

First-pass question types:
- compute from a standard formula
- choose the correct distribution/model
- derive an estimator or transformation
- identify conditions and degrees of freedom
- compare two similar formulas

Output style:
- Start with "what the problem is asking".
- Show the first line to write.
- Explain every new symbol once.
- Prefer one template per question type.
- Use source exercises over generated questions when visible.

High-risk failures:
- dumping formulas without conditions
- skipping the model-selection step
- assuming the student knows notation
- giving a plan without a problem-starting template

## Conceptual Understanding

Examples: computer organization, networks, operating systems, databases, economics, mechanical principles.

Default rescue artifacts:
- concept map with 4-8 nodes
- compare-and-contrast table
- process or mechanism card
- short-answer sentence frames
- common confusion list

First-pass question types:
- define and explain
- compare A vs B
- draw or interpret a process
- identify cause/effect
- choose the correct concept from a scenario

Output style:
- Explain the job of each concept before details.
- Use "if the question says X, answer with Y" frames.
- Include one compact process diagram only if readable.

## Memorization And Recall

Examples: politics, history, law foundations, many humanities general courses.

Default rescue artifacts:
- keyword compression sheet
- short-answer frames
- essay skeleton
- high-frequency recall drill

First-pass question types:
- list points
- explain a named theory
- match person/event/concept
- short essay or material analysis

Output style:
- Keep keyword order stable.
- Use "core sentence + 3 points + closing sentence" frames.
- Avoid long prose notes before the user has a memorization scaffold.

## Practical Skills

Examples: programming, CAD, experiments, engineering drawing, design tasks.

Default rescue artifacts:
- operation flow
- deliverable checklist
- common error fixes
- reproduce-one-sample task

First-pass question types:
- complete a task
- debug a common error
- explain an operation sequence
- produce a file, code, drawing, or report fragment

Output style:
- Give commands or steps in execution order.
- Include a "done when" checklist.
- If the user is blocked, repair the exact failing step first.

## Language And Expression

Examples: English, writing, translation, oral presentation.

Default rescue artifacts:
- question-type strategy
- sentence templates
- high-frequency vocabulary or phrase bank
- timed micro-practice

First-pass question types:
- reading comprehension
- translation
- writing
- cloze or grammar
- oral answer

Output style:
- Give reusable sentence frames.
- Include one before/after example.
- Practice under time limits once the template is clear.

## Open-Ended Synthesis

Examples: open-book exams, course papers, case analysis, project defense, comprehensive design.

Default rescue artifacts:
- answer framework
- material index
- reusable cases
- argument scaffold

First-pass question types:
- analyze a case
- propose a solution
- defend a design
- connect theory to material

Output style:
- Build an answer skeleton first.
- Include where to pull evidence from the user's materials.
- Do not over-polish before the structure works.
