# Rescue Output Patterns

Use these patterns in Rescue Teaching Mode. Adapt the labels to the course language, but keep the cognitive load low.

## Pattern A: Material-First Rescue Pack

Use when the user uploads materials but does not ask a precise question.

```text
What I Can See
- Files:
- Exam signals:
- Confidence:

Rescue Units
| Unit | Why It Matters | First Action | Practice Source | Skip For Now |
|---|---|---|---|---|

Start Here
1. Topic:
2. Plain meaning:
3. First line to write:
4. One micro-example:
5. Now practice:

I Need From You
- exam time or remaining study hours
- exam format
- target outcome
```

Do not include a multi-day plan unless remaining time is known.

## Pattern B: Symbol Decoder Card

Use before any formula-heavy table.

```text
Symbol Decoder
| Symbol In Material | Plain Meaning | What You Do With It |
|---|---|---|
| ... | ... | ... |

Tiny Rule
If you see ___, first write ___.
```

Bad: `MLE: theta=max Xi`.

Better: "Maximum likelihood means choosing the unknown number that makes the observed data most plausible. For a uniform range from 0 to theta, all data must fit inside the range, so the smallest legal theta is the largest observed sample value."

## Pattern C: Question-Type Card

Use when a topic maps to repeated exam tasks.

```text
Question Type: ...

How To Recognize It
- The problem says:
- The formula looks like:
- The answer usually asks for:

What The Examiner Wants
- ...

Do-This Template
1. ...
2. ...
3. ...

Common Traps
- ...

Practice Now
- Source exercise:
- If no source is available: generated mini drill:
```

## Pattern D: Micro-Example

Use one tiny example before assigning practice.

```text
Mini Example
Problem:

Step 1: ...
Why: ...

Step 2: ...
Why: ...

Answer shape:
```

The example may be partial if a full solution would be long. The goal is to show how to start.

## Pattern E: Wrong-Answer Repair

Use after the student attempts a problem.

```text
What You Did
- ...

Where It Broke
| Step | Issue | Fix |
|---|---|---|

Repair Drill
1. Redo only step ___.
2. Then do one similar problem:
3. Check yourself with:
```

Avoid shaming language. The repair should be precise enough to act on immediately.

## Visual Rules

- Prefer vertical cards over wide flowcharts.
- Keep each visual to one job: decode symbols, choose a question type, show steps, or diagnose mistakes.
- Put plain-language labels before formulas.
- If a diagram needs sideways scrolling, convert it to a table or numbered card.
