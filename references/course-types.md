# Course Types

Classify courses by how students earn exam points, not by major or department. Allow mixed percentages.

## Base Types

1. Calculation and derivation
   - Examples: calculus, linear algebra, probability, physics, circuits, mechanics.
   - Score sources: formulas, standard problem types, step marks, accurate calculation.
   - Sprint method: focus on typical problems, formula conditions, solution templates, and common mistakes.

2. Conceptual understanding
   - Examples: computer organization, computer networks, operating systems, databases, economics, mechanical principles.
   - Score sources: definitions, mechanisms, structure, flows, comparisons, diagrams, short answers.
   - Sprint method: build a framework, compare similar concepts, rehearse explanation templates.

3. Memorization and recall
   - Examples: ideological and political courses, modern history, law foundations, many humanities general courses.
   - Score sources: definitions, lists, theories, people, events, short-answer and essay frames.
   - Sprint method: compress to keywords, question-answer cards, high-frequency essay frames.

4. Practical skills
   - Examples: programming, CAD, SolidWorks, experiments, engineering drawing, design tasks.
   - Score sources: operation, code, debugging, experiment procedure, deliverables.
   - Sprint method: reproduce sample tasks, memorize operation steps, prepare common error fixes.

5. Language and expression
   - Examples: English, writing, translation, oral presentation.
   - Score sources: vocabulary, sentence patterns, reading strategies, writing templates, translation quality.
   - Sprint method: drill common question types, templates, high-frequency vocabulary, timed practice.

6. Open-ended synthesis
   - Examples: open-book exams, course papers, case analysis, project defense, comprehensive design.
   - Score sources: organization, argument structure, evidence, cases, clear expression.
   - Sprint method: build indexes, answer frames, reusable cases, and draft paragraphs.

## Example

Computer organization usually starts as:

- Conceptual understanding: 50%
- Calculation and derivation: 30%
- Practical skills: 20%

If labs, Logisim, Verilog, or assembly dominate assessment, increase practical skills.
