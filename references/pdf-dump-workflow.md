# PDF Dump Workflow

Use this when the user uploads many lecture PDFs, slides, notes, screenshots, or messy course files.

## Goal

Turn a pile of materials into a usable sprint map before creating detailed notes.

## Steps

1. Inventory
   - List files, likely chapters, page counts if available, and obvious gaps.
   - Note duplicate, unreadable, scanned, or poorly named files.

2. Extract structure
   - Find titles, headings, table of contents, summaries, formulas, examples, diagrams, assignments, and review questions.
   - Build a course module map.

3. Detect content density
   - Formula-heavy: likely calculation and derivation.
   - Diagram/process-heavy: likely conceptual understanding.
   - Definition/list-heavy: likely memorization and recall.
   - Code/procedure/report-heavy: likely practical skills.
   - Reading/writing-heavy: likely language and expression.
   - Case/project-heavy: likely open-ended synthesis.

4. Classify
   - Estimate course type mixture and tags.
   - Keep the classification adjustable if the user provides exam format or teacher emphasis.

5. Rank priorities
   - Use lecture emphasis, repetition, examples, summary pages, assignments, formulas, and foundational value.
   - Mark confidence based on available evidence.

6. Ask minimal questions
   - Exam time or remaining study hours
   - Exam format
   - Target outcome

7. Produce first pass
   - Material map
   - Course structure
   - Priority table
   - Missing information
   - Recommended sprint route

8. Generate study artifacts
   - Cram plan
   - One-page sheet
   - Definitions/formulas/process checklist
   - Self-test questions
   - Wrong-answer repair loop

9. Final contraction
   - Before the exam, produce a final 20-minute sheet, high-risk mistakes, must-remember items, and skip list.

## Confidence

High confidence requires at least two strong signals, such as teacher emphasis plus past papers, or assignments plus repeated lecture examples.

Medium confidence is normal for lecture-only PDF piles.

Low confidence means the output should be framed as a general rescue route rather than a teacher-specific prediction.
